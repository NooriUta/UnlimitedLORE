# LORE v1.0.55 — переезд на сервер и включённая защита

Два больших пласта, которые невозможно было разделить: инфраструктура уехала с ноутбука на постоянный сервер, и вместе с ней включилась настоящая аутентификация — раньше LORE верил заголовку `X-Seer-Role`, который мог прислать кто угодно.

61 коммит с `v1.0.54`. Спринты `SPRINT_LORE_CI_SERVER_MIGRATION` и `SPRINT_LORE_ADMIN_PANEL`.

## Переезд

ArcadeDB с `system_aida_lore`, Keycloak со своим realm `omilore`, Forgejo с раннерами, MinIO и сам стенд — всё переехало на ci-server. Адреса вынесены в переменные (`ARCADE_HOST`, `KC_HOST`, `FORGEJO_HOST`, `INFISICAL_HOST`): возврат на прежнюю топологию стоит строки в `.env`, а не правки compose.

Секреты уехали в Infisical. `SecretProvider` научился machine identity — сервисных токенов в свежих сборках больше нет. Учётка ArcadeDB сведена в один бин `MartCredentials`: раньше пароль инжектился в семь классов, и каждый собирал Basic-auth своей копией метода.

Токен `forgejo-mcp` понижен с административного до repo-scoped — сначала **измерено**, что админ для CI-диагностики не нужен, потом понижено.

## Аутентификация включена

`quarkus.oidc.enabled=true`, вход через Keycloak, роли из проверенного токена. Подделать `X-Seer-Role` больше нельзя: фильтр снимает клиентский заголовок и ставит роль из клейма.

**Чтение тоже закрыто.** Это оказалось важнее, чем выглядело: запись отбивалась, а `GET /lore/slice/*` отдавал весь граф анониму — 392 КБ по одному `curl`, без логина. Незаметно потому, что интерфейс вёл себя правильно: `AuthGate` просил вход, а API отдавал те же данные мимо него. **Защита интерфейса не является защитой данных.**

## Авторизация проверяется, а не декларируется

Вторая ось RBAC — клейм `agent_scope`. Восемь клиентов по профилям; узкий профиль получает `AGENT_SCOPE_FORBIDDEN` при записи в чужое семейство.

Проверено парной пробой, а не одиночной: два сервисных аккаунта с **одинаковыми** ролями на одном запросе — `agent-full` проходит, `agent-marketer` отбивается. Одиночная проверка не различила бы «отсекает по скоупу» и «сломано у всех» — а именно это состояние было часом раньше, когда клейм приходил в кавычках и отсекались все агенты, включая полный.

Закрыты три семейства вне матрицы (`forgejo`, `asset`, `quality-gate`) — под ними была живая запись, и мерж PR мог сделать любой профиль.

## HTTPS в локальной сети

Вход по OIDC невозможен на `http://<ip>:4400`, и это свойство браузера, а не настройка: `crypto.subtle` существует только в защищённом контексте, а `oidc-client-ts` использует его для PKCE. Пока стенд жил как `http://localhost:4400`, всё работало — localhost единственное исключение спецификации. Переезд сменил адрес и молча отобрал криптографию.

`lore.odal.seidrstudio.pro` за Traefik, сертификат Let's Encrypt по DNS-01 — наружу при этом ничего не открывается.

Сетевого ограничения через `ipAllowList` в релизе **нет намеренно**: на этом хосте оно не способно различать клиентов. Порты публикуются через docker-proxy, и до middleware доезжает один и тот же внутренний адрес для всех — список либо закрывает доступ всем, либо пропускает всех, выглядя работающей защитой.

## CI/CD

Публикация Release по тегу (её не было — пять тегов уехали без объектов Release), передача прокси в сборку образа, повторы на всех шагах с сетью, Node и JDK из образа раннера вместо загрузки, кеширующее зеркало Nexus для Maven, npm и docker-образов.

Зеркало **опционально**: неудача повторяется без него. Список запасных репозиториев для этого не годится — Gradle идёт за `.jar` туда же, откуда взял `.pom`, и повторного поиска не делает.

## Клиент-ловушка отключён

В realm существовал клиент `lore-mcp` — по имени «клиент для MCP», и именно на него вела прежняя инструкция. Маппера `agent_scope` у него нет, поэтому токен приходил бы без клейма, фильтр счёл бы вызывающего не-агентом и пропустил всё. Авторизация выглядела бы включённой, а главный потребитель шёл бы мимо неё.

Его секрет так и оставался placeholder'ом — то есть клиентом никогда не пользовались, и это было верно. Клиент отключён: подключиться нельзя, история сохранена.

## Уборка

Удалены 17 пустых отменённых спринтов, 51 осиротевшая строка истории и снятый пласт `PlanItem` — 1470 вершин и 1963 ребра, которых не читал ни один слайс.

## Известные ограничения

- Публичного доступа нет: адрес резолвится в локальную сеть, учётка в realm пока одна.
- Слой MUNINN (`/bench/*`) аутентификацией не закрыт — отдельное решение.
- У сервисных аккаунтов узких агентов realm-роль `admin`; их ограничивает только `agent_scope` (`AL-66`).
