# LORE v1.0.50 — 🎉 юбилейный: Admin LORE перестаёт врать и обретает структуру

Спринт SPRINT_LORE_ADMIN_PANEL, фазы SAFETY/UI/TESTS (PR #146, #147). База — UX-разбор
владельца и системная экспертиза: 15 находок стали задачами AL-31…AL-48, спека сценариев
SPEC-ADMIN-LORE-UC 1.1.0 (многопользовательская модель: соавторы + demo-viewer).

## 🛡 PHASE_SAFETY — защита от потери доступа (AL-35, AL-45)
- **Guard от self-lockout**: старт с `LORE_AUTH_ENABLED=true` при нуле включённых
  admin/super-admin → отказ загрузки с инструкцией отката; снятие роли с последней
  администрирующей учётки → **409 LAST_ADMIN**. Инвариант живёт в бэкенде (KcBridge +
  LoreAuthStartupGuard), UI лишь показывает.
- `GET /lore/kc/auth-preflight` — живой предполётный чеклист включения auth.
- `GET /lore/kc/denials` — последние 401/403/409 по /lore/* (кольцевой буфер, UC-A7
  реактивный минимум; долговременный аудит — AL-20).

## 🧭 PHASE_UI — панель по одобренному прототипу (AL-32/34/36/38/40/41/42/46)
- **Навигация по риску**: Доступ (--dng) / Справочники (--acc) / Система (--inf),
  счётчики, крошки; вкладка и карточка адресуются URL (`&tab=`, `&user=`, `&dict=`).
- **Четыре состояния KC-моста различимы** (AL-31): пусто ≠ 403 ≠ 503 ≠ сеть.
- **«Роли и права»**: обе оси + обратная матрица «Кто имеет доступ к…» (human-only
  строки §4 подсвечены); плашка «права пока не проверяются» до AL-17 — экран не
  утверждает защиту, которой нет.
- **Карточка человека**: роли на месте, guard-врезка, честное «истории пока нет».
- **Словари**: code read-only (ключ!), «использований: N», пикер цвета из токенов темы,
  иконка с подсказками по всей коллекции game-icons и живым превью чипа; невалидный
  slug виден ДО сохранения — пустые иконки area больше не судьба.
- **Настройки**: реестр известных ключей (честно пуст), «Неопознанные значения»
  с удалением, чеклист auth, последние отказы.
- Единый тулбар поиск/счётчик/фильтр на всех вкладках; 93 тега теперь ищутся.
- Мобильный 375px: все 7 вкладок без горизонтального скролла.
