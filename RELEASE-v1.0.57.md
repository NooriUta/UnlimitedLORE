# v1.0.57 — сквозной поиск, живая аутентификация и цепочка develop → preprod → main

Первый релиз, срезанный по новой схеме: `develop` → `preprod` → `main` + тег. До него стенд обновлялся с каждого коммита `develop`, и проверка одной фичи уезжала под другую прямо во время проверки.

## Сквозной поиск по всему корпусу

Появился `GET /lore/search` и экран поиска: ранжирование BM25 × приоритет типа, фасеты по типу, компоненту и проекту, русская морфология и поиск по телам, а не только по заголовкам. Тот же поиск доступен агентам инструментом `search` (SRCH-01, SRCH-05, SRCH-07).

Заодно вылечен легаси-индекс: ретайр старого full-text вынесен на каждый старт. Внутри `javaStep` он на проде **не выполнялся никогда** — шаг помечался выполненным в ledger, а работа не делалась.

## Аутентификация доведена до рабочего состояния

Все правки этого блока родились из одного прод-инцидента 2026-07-21: UI показывал белый экран без выхода.

- **AL-69** — протухшая сессия больше не выглядит действительной. `getUser()` отдаёт пользователя даже с истёкшим сроком; приложение считало вход состоявшимся, рендерилось, а каждый запрос получал 401. Теперь протухшего сначала пробуют продлить, при неудаче — считают, что сессии нет.
- **AL-71** — заход по голому IP уводит на HTTPS-хост. Вход по OIDC на `http://<ip>:4400` невозможен в принципе: PKCE считает `code_challenge` через `crypto.subtle`, а браузер даёт этот API только в защищённом контексте. Редирект стоит в документных `location`, а не на уровне `server`: иначе он накрыл бы `/lore/` и `/bench/`, и MCP получал бы 301 вместо данных.
- **AL-74** — экран входа вместо молчаливого редиректа. Автоматический переход плох ровно в самом частом случае: сессия истекла посреди работы, человека выбрасывает без объяснения, а если Keycloak недоступен — он остаётся на чужом домене без пути назад. Он же был причиной «странного выхода»: после `signoutRedirect` приложение возвращалось на `/`, видело «сессии нет» и уводило обратно в Keycloak — выйти *в приложение* было нельзя.
- **AL-72** — audience `lore-api` вместо дефолтного `aud=account`. Раскатано на живой Keycloak одним client scope на десять клиентов. Валидация `quarkus.oidc.token.audience` **сознательно не включена**: уже выданные токены аудитории не несут.
- **AL-73** — тема логина Keycloak в оливковой палитре LORE, чтобы переход «наш экран → форма KC» не читался как «меня увели не туда».
- **AL-75** — закрыт пробел, из-за которого путь с включённой аутентификацией не проверялся ни одним тестом: все проверки шли по ветке «выключено». Тесты подтверждены мутацией.

## Инфраструктура

Цепочка `develop → preprod → main` с зеркалированием тега в GitHub. Промоушн на стенд идёт автономно, релизный тег — только по явному решению владельца. Тег обязан содержаться в `main`, иначе зеркалирование пропускается с предупреждением: релиз, срезанный мимо `main`, — ошибка процесса, а не вариант.

MCP переведён с `http://192.168.3.131:4400` на публичный HTTPS-хост: по локальному адресу шифрования не было, а публиковать его наружу означало бы гонять Bearer-токен по интернету открытым текстом.

Ретрай сборки образа перестал врать про причину: детерминированные отказы больше не помечаются «transient network» и не съедают три попытки (MIG-25).
