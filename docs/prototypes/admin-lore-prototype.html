Люди

Учётные записи Keycloak и их realm-роли. Пароли задаются в Keycloak — LORE их не принимает и не хранит.

Прототип · показать состояние:

Ни одного человека ещё не заведено

Keycloak отвечает нормально, в realm omilore просто нет учётных записей. Заведите себя первой — до включения аутентификации, иначе войти будет некому.

GET /lore/kc/users → 200 · []

Нет прав смотреть список людей

Ваша роль — viewer. Управление учётными записями доступно только роли admin. Это не пустой список: люди могут существовать, но они вам не видны.

GET /lore/kc/users → 403 · admin role required

Keycloak не подключён

Мост в Keycloak не настроен, поэтому список людей получить неоткуда. Это не значит, что учётных записей нет — их статус сейчас неизвестен.

GET /lore/kc/users → 503 · KC_NOT_CONFIGURED

Включение аутентификации

Включить сейчас нельзя: в системе нет ни одного человека с ролью admin. Аутентификация отключит доступ по конфигу — войти станет некому, и вернуться можно будет только через консоль Keycloak.
✕Есть хотя бы один человек с ролью adminСейчас: 0. Заведите себя на вкладке «Люди» и назначьте admin.
✓Keycloak доступенМост отвечает, realm omilore, 8 клиентов агентов на месте.
✓Клиент lore-app настроен на PKCEВход людей — Authorization Code + PKCE.
✕Права агентов проверяются на бэкендеAgentScopeFilter ещё не включён (AL-17): после включения auth агенты пройдут в любое семейство.

Что здесь изменено против текущей панели
  • Навигация — по риску, не по таблицам. «Доступ» (люди, агенты, права) отделён от «Справочников»: выдача admin и правка подписи больше не равны по весу.
  • Небезопасное состояние кричит, а не шепчет в углу серым.
  • Три разных пустых состояния вместо одного «заведите первого»: пусто, отказ, мост недоступен — переключатель на вкладке «Люди».
  • Обратная матрица «кто имеет доступ к X» — вопрос, который задают на ревью доступа.
  • Карточка человека с историей ролей и действий — место, куда ляжет аудит.
  • Код read-only, «использований: N» у каждого значения, цвет и иконка — выбором с живым превью, а не вводом вслепую.
  • Настройки — реестр известных ключей с типом и описанием; чужое собрано отдельно и удаляется.
  • Включение auth — предполётный чеклист с блокировкой, пока нет ни одного админа.
