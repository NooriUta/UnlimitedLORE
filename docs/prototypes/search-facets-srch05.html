SRCH-05 · плашки и фасеты

Поиск с фильтрами по типу, компоненту и проекту

Живые данные прода. Плашки типов — мультивыбор, как в фильтрах списков. Компонент и проект берутся из связей, а где связи нет — выводятся из родителя, иначе фильтр молча съел бы половину корпуса.

уходит в индекс как
    Под эти фильтры ничего не попало.

    Что показал этот прототип

    • Голый префикс ломает русский. «миграция*» давала 0 при 55 по слову: индексируется основа «миграц», а wildcard Lucene не анализирует. Теперь каждое слово идёт как (слово OR слово*) — объединение строго не хуже: «токен» 15+10 → 19.
    • Связей на компонент почти нет там, где больше всего данных. У задач 0 из 2847, у решений 1 из 292. Фильтр «в лоб» скрыл бы их целиком. Здесь компонент выводится: задача → спринт из task_uid, решение → родительский ADR. Выведенные связи помечены курсивом.
    • Проект почти не проставлен. Есть у спринтов (337) и PR (334), у спек и задач — ноль. Фильтр по проекту полезен, но пока узкий.
