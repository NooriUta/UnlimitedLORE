# LORE v1.0.54 — переезд на ci-server: инфраструктура, которая до этого держалась на ноутбуке

Первый релиз, где содержимое — не функциональность, а фундамент. LORE, его база, Keycloak, Forgejo и раннеры уехали с ноутбука разработчика на постоянный сервер. Спринт `SPRINT_LORE_CI_SERVER_MIGRATION`.

Функциональность продукта в этом релизе почти не менялась. Менялось то, без чего она не работает.

## Что переехало

ArcadeDB с `system_aida_lore`, Keycloak вместе со своей базой и realm'ом `omilore`, Forgejo с раннерами, сам стенд LORE. Адреса вынесены в переменные (`ARCADE_HOST`, `KC_HOST`, `FORGEJO_HOST`, `INFISICAL_HOST`) — возврат на прежнюю топологию стоит одной строки в `.env`, а не правки compose.

Заведена схема поддоменов `odal.seidrstudio.pro`; Keycloak опубликован через Traefik с сертификатом Let's Encrypt по DNS-01.

## Чего переезд стоил — семь отказов, каждый со своим обманчивым симптомом

Общее у них одно: **симптом указывал не туда, где была причина.** Это и есть главный урок релиза.

**CD не пережил переезд по трём независимым причинам сразу.** Windows-пути томов (`C:/AIDA/...`) валидны на ноутбуке и отвергаются Linux-демоном. У job'а нет `.env`, поэтому деплой поднял бы бэкенд с пустыми секретами мостов — зелёный деплой при мёртвом KC-мосте. Имя compose-проекта выводилось из каталога раннера, то есть рядом с живым стендом вырос бы второй.

**Запись из интерфейса отваливалась, а чтение работало.** Браузер не шлёт `Origin` на GET, поэтому CORS резал только POST — «интерфейс открывается, кнопки не работают». Адрес стенда сменился при переезде, а список origin'ов остался прежним.

**«MCP не отвечает» имел четыре причины**, и каждая по отдельности объясняла симптом неполностью: конфиг в user-скоупе перекрывал проектный; локальный прокси резал http-серверы; бэкенд был забинжен на `127.0.0.1` и недостижим по LAN; и настоящий баг в `forgejo-mcp` — один транспорт на процесс, второй клиент всегда получал `400 Server already initialized`. Четыре рестарта подряд не помогали, потому что диагностическая проверка сама занимала единственный слот.

**Статус `deferred` в реестре открытых вопросов не работал никогда.** `trigger` — зарезервированное слово SQL ArcadeDB, запись падала на разборе, а валидация не пускала `deferred` без триггера. Замкнутый круг, невидимый, пока никто не пробовал отложить вопрос по-настоящему: вопросов с заполненным триггером в корпусе было ноль.

**Стек поднимался с `restart: no`** — после перезагрузки хоста база вставала, а приложение нет. Симптомом было бы «MCP не отвечает», и диагностика ушла бы в сторону, которая уже стоила дня.

**Публикации релизов не существовало вовсе.** Пять тегов были запушены без объектов Release; страница показывала состояние двухдневной давности. Пока стенд жил на ноутбуке, ручная публикация была нормальной — с появлением сервера она стала шагом, который никто не выполняет.

## Секреты уехали в Infisical

`SecretProvider` научился machine identity (Universal Auth): логин по client-id/secret, кэш access-токена, перелогин при 401. Сервисные токены в свежих сборках Infisical убраны, так что это единственный путь.

Учётка ArcadeDB сведена в `MartCredentials` — раньше пароль инжектился в семь классов, и каждый собирал Basic-auth своей копией метода. Два источника по приоритету: секрет-сервис → значение конфига. Запас намеренный: в отличие от секретов мостов, пароль БД несущий, и недоступный Infisical не должен ронять старт.

## Права доступа Forgejo понижены

Токен `forgejo-mcp` был административным. Сначала **измерено**, что для CI-диагностики админ не нужен вовсе, потом понижено до repo-scoped. Прежний токен удалён; среди токенов не осталось ни одного с `write:admin`.

Добавлена раздельная идентичность вызывающей стороны: путь определяется тем, **какой ключ предъявлен**, а не заголовком — заголовок подделал бы любой, у кого есть прямой ключ, и аудит стал бы фикцией.

## CI/CD

Публикация Release по тегу (её не было), проба доставки build-args, трансляция прокси для Gradle на раннере (Gradle не читает `HTTP_PROXY` из окружения — только системные свойства Java), политики перезапуска контейнеров, свой CI/CD для `forgejo-mcp` с регрессией на многосессионность.

## Уборка

Удалены 17 пустых отменённых спринтов, 51 осиротевшая строка истории и снятый пласт `PlanItem` — 1470 вершин и 1963 ребра, которых не читал ни один слайс. Синхронизированы `main` и `develop`, разошедшиеся со времён GitHub.

## Что осталось за границей релиза

Аутентификация подготовлена, но **не включена**: флаги, клиенты KC, маппер ролей и пользователь готовы, связка проверена локально против боевого Keycloak. Включение и вторая ось RBAC (`agent_scope` проверяется, а не декларируется) — следующим шагом.

Публикация LORE наружу и `ipallowlist` на админку Keycloak — решения владельца, вынесены отдельными задачами.
